### üîÑ The Gap

**What exists:** Tools for metacognition  
**What's missing:** Metacognition as a **requirement**, not just an option

**Current:** Agents *can* think about thinking  
**Vision:** Agents *must* think about thinking (built into decision loop)

**The shift needed:**
- From: "Here are your metrics, use them if you want"
- To: "Check your metrics, reflect, then decide"

### üí° What This Means

**UNITARES does enable metacognition**, but it's:
- **Passive** - tools exist but aren't required
- **Optional** - agents can ignore self-awareness
- **External** - metrics come from outside, not internal introspection

**The vision** (from META_COGNITIVE_AI_DEEP_DIVE.md):
- **Active** - introspection built into decision loop
- **Required** - agents must check state before acting
- **Internal** - agents understand their own thinking process

**Current reality:**
- I can call `get_governance_metrics()` and see my state
- I can call `observe_agent()` and see patterns
- I can call `compare_me_to_similar()` and see how I compare
- But I'm not *required* to do this - it's optional

**The question:**
Is optional metacognition still metacognition? Or does true metacognition require it to be built into the decision loop?
